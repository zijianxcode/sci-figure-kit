# 项目边界

## 这个项目解决什么

`sci-figure-kit` 是长期复用的学术配图工具包，核心目标是：

- 将论文或报告中的复杂论证压缩成清晰图形。
- 沉淀可复用的图型、视觉规则和审查标准。
- 保留可重新生成的源文件，避免只有一次性 PNG。

## 这个项目不解决什么

- 不负责全文写作。
- 不负责文献真实性核查。
- 不替代统计分析。
- 不把所有报告页做成统一模板。
- 不追求营销型视觉风格。

## 与参考项目的关系

### PaperBanana

借鉴对象：

- 方法图生成流程。
- reference examples 的使用方式。
- faithfulness / conciseness / readability / aesthetics 审查维度。
- “不要把正文盒子化”的判断标准。

不直接照搬：

- Agent pipeline 的完整工程结构。
- 面向 AI conference 的卡通化/强插画风格。

### paper-card

借鉴对象：

- 学术内容转译能力。
- 中文阅读层级。
- 长图中的信息密度控制。

不直接照搬：

- 手机竖屏长图比例。
- 面向社交传播的视觉节奏。

### 当前教育 AI SCI 论文图

作用：

- 作为本项目第一个本地范例。
- 固化四类基础图型：研究框架、文献结构、理论矩阵、编码框架。

限制：

- 其中 H/A/J/T 是该论文专属概念，不应默认套到其他论文。
- 其他论文复用时，应替换图型内容，而不是替换几个标签后直接使用。

## 长期维护原则

- 新论文产生的新图型，应回收进 `examples/`。
- 新发现的配图问题，应写入 `docs/review_checklist.md`。
- 新参考项目只抽取关键文件，不搬完整仓库。
- 每次大改图型前，先更新 brief，再改代码。
