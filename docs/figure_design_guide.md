# 论文配图设计指南

## 1. 先定义图的任务

每张图先写一句任务句：

> 读者看完这张图后，应理解什么关系？

如果一句话说不清，图的范围通常太大。论文图不负责复述全文，只负责压缩一个关键关系。

## 2. 四类常用图

### 研究框架图

适合表达研究问题、假设、变量和实证链条之间的关系。

不要把研究背景、文献缺口、方法设计、贡献全部放进去。那些内容属于正文。

### 文献结构图

适合表达文献线索如何汇聚到研究缺口。

如果正文已经说明“漏斗只是启发式组织方式”，图上要避免画成严格线性下沉。

### 理论适配矩阵

适合比较多个理论框架对某一研究问题的解释能力。

若没有明确评分方法，避免 1-5 雷达图。用“强 / 中 / 弱 / 需补充”等定性判断更稳。

### 编码框架图

适合表达编码单元、编码状态和特殊维度之间的关系。

不同层级不能画成并列项。例如 H/A/J 是状态，T 是切换指标，应分开画。

## 3. 图内文字规则

- 使用关键词和短语，不放完整段落。
- 不把完整图注画进图片。
- 图题尽量由正文排版系统处理，图片内部只保留必要的模块标签。
- 图中文字应能在 A4 宽度下辨认。

## 4. 视觉规则

- 使用浅色分区表达逻辑组，颜色服务于结构。
- 保持 16:9 或 4:3 的紧凑矩形，不产生大量空白。
- 同一类元素使用同一形状和尺寸。
- 箭头要有语义：主流程、依赖关系、反馈、动态切换不能全部用同一种线。
- 饱和色只用于关键贡献、风险或特殊维度。

## 5. 当前论文的固定语义

- H：人类主导，学习者保留关键决策权。
- A：AI 输出实际占据，行为层面采纳 AI 输出，缺少显式协商。
- J：共同协商，多轮对话后共同确定。
- T：动态切换指标，记录 H/A/J 在节点内或跨节点之间的切换频率与方向。
