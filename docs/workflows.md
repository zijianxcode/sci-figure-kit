# 复用工作流

这个项目用于两类学术场景：

- 论文正文配图：强调严谨、可验证、低噪音。
- 学术报告配图：保留论文逻辑，但允许更强的讲述性和视觉节奏。

## 论文正文配图

1. 先写图的任务句。

   示例：读者看完这张图，应理解 RQ1 如何支撑 RQ2 与 RQ3。

2. 填写 `templates/figure_brief.md`。

3. 判断图型：

   - 研究框架图：变量、研究问题、假设、实证链条。
   - 文献结构图：多条文献线索汇聚到缺口。
   - 理论适配矩阵：比较理论解释力与操作化能力。
   - 编码框架图：编码单元、编码状态、特殊指标。
   - 统计图：基于真实数据展示结果，不用示意图替代。

4. 先画结构草图，不先调颜色。

5. 使用 `docs/review_checklist.md` 审查。

6. 导出 SVG 进入 Figma 做微调，保留可编辑源。

7. 导出 PNG 后嵌入正文，检查 A4 宽度下是否可读。

## 学术报告配图

报告图可以比论文图更叙事，但不能牺牲概念准确性。

建议做法：

- 每页只讲一个关系。
- 论文正文中的复杂矩阵可以拆成 2-3 页逐步出现。
- 可以保留更大的字号和更少的元素。
- 可以使用更强的色彩强调，但同一报告内要保持色彩语义一致。
- 不把正文图原样塞进 PPT。论文图适合审稿人，报告图适合听众。

## 从论文图转报告图

| 论文图 | 报告图处理 |
|---|---|
| 研究框架图 | 拆成“研究问题链条”和“假设分组”两页 |
| 文献结构图 | 先讲三条文献线索，再讲汇聚到缺口 |
| 理论适配矩阵 | 只保留关键对比列，隐藏次要列 |
| 编码框架图 | 先讲 H/A/J，下一页单独讲 T |
| 统计图 | 保留数据，减少网格和注释 |

## 每次新项目的目录建议

```text
outputs/
└── <project-name>/
    ├── briefs/       # 每张图的 brief
    ├── html/         # 图源文件
    ├── png/          # 导出图
    └── review/       # 检查记录
```

## 复用时的最低标准

- 每张图有 brief。
- 每张图有明确图型。
- 每张图通过 checklist。
- 图名、正文引用、图片文件名一致。
- 可重新生成，不只保存最终 PNG。
- 每张图保留 Figma 可编辑 SVG。
